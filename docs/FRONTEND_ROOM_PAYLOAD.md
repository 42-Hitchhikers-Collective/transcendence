# Frontend Room Payload Documentation

This document describes the exact payloads that are broadcasted to frontend clients via the `room_state` event.

The payloads are generated by `getFrontendRoom(room, observerPlayerId)` in `gameManager/dataToFrontend.ts`.

---

## Key Security Feature

Each player receives a **customized version** of the room data based on their own `playerId`:
- **Their own cards**: Visible only to themselves
- **Other players' cards**: Hidden (only card count shown)
- **isTheObserver**: Flags which player in the list is "you"

---

### What the Frontend Can Do
- Render **your hand** (only visible because `isTheObserver: true`)
- Render **opponent cards as face-down** with only the count visible

- Display **draw pile count**
- Enable click handlers on your own cards to play them
- Disable interactions with other players' cards

## Key Field Descriptions

| Field | Type | Description | Always Present |
|-------|------|-------------|-----------------|
| `id` | string | Room ID (unique) | ✅ Yes |
| `state` | "waiting" \| "playing" \| "finished" | Current room state | ✅ Yes |
| `players` | array | Array of FrontendPlayer objects | ✅ Yes |
| `game` | object \| undefined | Game state (null in waiting) | ❌ Only when playing |
| `players[].id` | string | Player's unique ID | ✅ Yes |
| `players[].userName` | string | Display name | ✅ Yes |
| `players[].isTheObserver` | boolean | `true` = this is YOU | ✅ Yes |
| `players[].isReady` | boolean | Ready to start/playing | ✅ Yes |
| `players[].cardCount` | number | How many cards in hand | ✅ Yes |
| `players[].cards` | array \| undefined | Actual card objects | ❌ Only if observer + playing |
| `game.currentPlayerId` | string | Whose turn it is | ❌ Only when playing |
| `game.discardTopCard` | object \| null | Top card on discard pile | ❌ Only when playing |
| `game.drawPileCount` | number | Cards left in draw pile | ❌ Only when playing |

## Broadcast Events

### When is `room_state` emitted?

The `room_state` event is broadcasted in the following situations:

1. **Player joins room** → `registerRoomHandlers` → `socket.on("join_room")`
2. **Player leaves room** → `registerRoomHandlers` → `socket.on("leave_room")`
3. **Game starts** → `registerGameHandlers` → `socket.on("start_game")` or auto-start when all ready
4. **Player plays a card** → `registerGameHandlers` → `socket.on("play_card")`
5. **Player draws a card** → `registerGameHandlers` → `socket.on("draw_card")`
6. **Player reconnects** → `registerConnectionHandlers` → auto-rejoin room
7. **Player ready status changes** → `registerGameHandlers` → `socket.on("set_ready")`

Each event calls `broadcastRoomState(roomId)` which sends the customized payload to each player.

---

## Example Broadcast Flow

```
User clicks "Ready"
    ↓
socket.emit("set_ready", { isReady: true })
    ↓
backend: gameManager.setReady(playerId, true)
    ↓
backend: broadcastPlayerState(playerId) [emits "player_state" event]
    ↓
backend: if (allPlayersReady) gameManager.startGameAuto(playerId)
    ↓
backend: broadcastRoomState(roomId) [emits "room_state" with game object]
    ↓
frontend: socket.on("room_state", (frontendRoomData) => {
    // Now frontendRoomData.state === "playing"
    // frontendRoomData.game is populated
    // frontendRoomData.players[myIndex].cards contains my hand
})
```

## Type Definitions (TypeScript)

```typescript
export type FrontendPlayer = {
  id: string;
  userName: string;
  isTheObserver: boolean;
  isReady: boolean;
  cardCount: number;
  cards?: { color: string; value: string | number }[];
};

export type FrontendRoom = {
  id: string;
  name: string;
  state: "waiting" | "playing" | "finished";
  players: FrontendPlayer[];
  game?: {
    currentPlayerId: string;
    discardTopCard: { color: string; value: string | number } | null;
    drawPileCount: number;
  };
};
```
